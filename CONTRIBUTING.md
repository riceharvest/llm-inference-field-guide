# Contributing

Contributions must be reproducible and must distinguish measured results from derived estimates and hypotheses.

For benchmark claims, include hardware, software commit/version, exact model revision/file, full command, prompt/output/context, warm-ups, every repeated sample, median, memory, and output-validity result. Never submit only the fastest run.

Keep private paths, usernames, tokens, API keys, logs containing prompts, and machine state out of commits. Use generic paths in documentation.
